1. 架构总览

本版本将版本管理能力从 cmd/arch-manager/version.go（528 行单体文件）下沉为 go-core 通用版本管理模块，实现前后端全覆盖的版本管理体系。

go-core 新文件
8 个
cmd/arch-manager 重构
2 个文件
前端增强
1 个文件
新增 API 端点
5 个
测试用例
≥ 60
Benchmark
≥ 8

设计原则

融合三大模式：release-please 的 Release PR 模式 + changesets 的变更意图文件 + semantic-release 的插件化架构，全部 Go 原生重实现。

核心约束兼容：所有新模块通过 C1-C6 六条核心约束验证，版本管理模块以 Adapter[In, Out] 模式接入架构。

通用性：版本管理模块不绑定任何特定项目结构，通过接口抽象支持任意 Go 项目的版本管理。

类型VersionBump — 版本号增量（Major/Minor/Patch/Prerelease）
类型ReleaseStep — 发布流水线步骤（Step[ReleasePlan, ReleasePlan]）

T08 发布流水线编排器

  2. InferVersion — 推断下一版本号
  3. MergeChanges — 合并 ArchChange 变更意图
  4. GenerateChangelog — 生成 Changelog
  5. UpdateADR — 更新/创建 ADR
  6. CreateTag — 创建 Git Tag
